# Spec 0004 — bridge: Session controller, AdapterFactory, wiring, echo

Implementation contract for ROADMAP lane 1, entry 7 (session B, its last
headless entry). First spec written under the revised process: it is authored
on the entry's branch, agreed in conversation before any code, and merges with
the implementing PR. The entry is delivered as **two PRs** (7a and 7b — see
the delivery plan below); this one spec covers both, and each PR is judged
against its slice.

## Goal

Everything built so far is a collaborator waiting for its orchestrator. This
entry adds the **Session controller** — the state machine that runs a whole
bridge session from handshake to teardown — plus the **AdapterFactory** port
that defers mode-specific construction until `hello` reveals the mode, and
`wiring.py`, the composition root. After this entry, the only code missing
from a working bridge is the NVDA-facing adapter layer (session C): the domain
is complete and fully exercised headlessly.

The controller owns the bridge's core promise (AGENTS.md hard invariant 3):
**every teardown path restores the user's synth**. A crashed harness, a dead
socket, a forgotten session — none of them may leave a blind user with a mute
screen reader. That promise is proven here with fakes, on every path, before
any real NVDA code exists.

Second goal, agreed in conversation: this entry deliberately produces the
**headless integration surface lane 2 will test against**. Because the Session
is pure, the *real* Session — real dispatch, real framing, real watchdogs —
can run on a thread with fake NVDA-facing adapters and answer over the wire
exactly like a live bridge. The scriptable fakes and the wire-level scenario
in PR 7b are that surface: the server's integration tests reuse it instead of
talking only to a scripted fake bridge.

## Wire change — the `echo` command

The one addition to the shared protocol, additive to v1 (no version bump; on
an older bridge it fails like any unknown command, which is the tolerant
behaviour this spec already requires):

- `Command.ECHO = "echo"`; `EchoParams(payload: Any)`;
  `EchoResult(payload: Any)`. The Session returns the payload unchanged.
- Why it earns a place on the production wire: it round-trips an *arbitrary*
  payload through the full stack — encode → frame → decode → validate →
  dispatch → re-encode — which is the one path no fake can prove and no
  speech-dependent command can isolate. Unicode, nesting, floats, long
  strings: framing edge cases become one assertion in an integration test.
- Canonical change in `shared/nvda_mcp_wire/protocol.py` + `shared/tests`;
  the addon's copy is regenerated by `sync_shared.py` as usual.

## New domain ports

One file per port; a port's DTO / signalling types live in its file. All are
`abc.ABC` with `@abstractmethod`. Real implementations arrive in session C
(NVDA adapters); this entry ships only fakes.

1. `domain/ports/adapter_factory.py` — **AdapterFactory**: `build(mode:
   CaptureMode) -> AdapterSet`. Called by the Session exactly once, after a
   successful `hello`, never before — mode is only known then (AGENTS.md
   **Decided**: no configure-after-construction). **AdapterSet** is a frozen
   dataclass in the same file bundling the mode-specific collaborators:
   `speech_source`, `braille_source`, `synth_swapper`, `gesture_sender`.
2. `domain/ports/speech_source.py` — **SpeechSource**: `start(buffer:
   SpeechBuffer) -> None`, `stop() -> None`. Whatever feeds the speech buffer
   (silent: the spy synth's `post_speech`; live: `pre_speechQueued`). `stop`
   is idempotent and never raises.
3. `domain/ports/braille_source.py` — **BrailleSource**: same shape over the
   `BrailleBuffer` (`braille.pre_writeCells` in production; both modes).
4. `domain/ports/synth_swapper.py` — **SynthSwapper**: `current_synth() ->
   str` (for `hello`'s reply in either mode), `swap_to_spy() -> str`
   (performs the whole silent-mode defense of RFC 0001's fail-safe section —
   config agreement, save guard, `getSynthInstance` patch — and returns the
   real synth's name), `restore() -> None` (reverses everything; idempotent —
   safe to call when nothing was swapped, safe to call twice). The **Session
   calls `restore()` in `finally` on every path unconditionally**; idempotency
   is what makes that unconditional call safe.
5. `domain/ports/gesture_sender.py` — **GestureSender**: `press(gesture_id:
   str) -> None`, blocking until NVDA has processed the gesture. Raises
   **GestureError** (same file) for an unknown/rejected id.

## The Session controller and its command handlers

**Amended during 7a implementation (rides in the PR, per process).** The first
cut had one `Session` class holding both the lifecycle *and* a flat dispatch
table of `_handle_*` methods. In conversation we agreed to split dispatch out:
each command becomes a small controller (a **`CommandHandler`**), the Session
becomes a thin dispatcher, and the two poll loops collapse into one. This
section describes the delivered shape; the invariants (handshake strictness,
the watchdogs, restore-on-every-path) are unchanged — only their housing is.
The full rule set is now **Decided** in AGENTS.md ("Command handlers").

### Lifecycle — `domain/controllers/session.py`

**Session** owns only the lifecycle. Companion `SessionConfig` (frozen: 
`nvda_version`, `heartbeat_timeout = 30.0`, `inactivity_timeout = 120.0`) rides
in the same file; **TeardownReason** (plain `enum.Enum`: `CLIENT_BYE`,
`CHANNEL_CLOSED`, `HEARTBEAT_TIMEOUT`, `INACTIVITY_TIMEOUT`, `HANDSHAKE_FAILED`,
`EXTERNAL`) moved to its own `teardown_reason.py` to break an import cycle.
Constructed with ports/config only — `MessageChannel`, `Transcript`, `Clock`,
`SessionConfig`, and the **command registry** — it no longer knows the
`AdapterFactory` (that lives in the hello handler). It builds one
**`SessionContext`** (the per-session bundle handed to every handler, with the
one lifecycle capability `close(reason)` wired to `request_teardown`). `run()`
executes on the caller's thread.

**One dispatch loop** — `while self._reason is None:` over a `_State`
(`PRE_HELLO` / `ESTABLISHED`). Each turn: absorb any external teardown request;
`read_message()`; a `TIMEOUT` just checks the deadlines and polls again;
`ChannelClosed` → `CHANNEL_CLOSED`; an unreadable line is `HANDSHAKE_FAILED`
pre-hello or a transcript note when established. A message is parsed, dispatched,
and the deadlines re-checked. Every exit sets `self._reason`; nothing else ends
the loop.

- **Watchdogs.** Any inbound message resets the heartbeat; only a command with
  `resets_inactivity` (everything but `ping`) resets inactivity. A lull → 
  `HANDSHAKE_FAILED` pre-hello, else `HEARTBEAT_TIMEOUT`; command silence when
  established → `INACTIVITY_TIMEOUT`. `request_teardown(reason)` is thread-safe
  (session C's plugin terminate / panic gesture).
- **Dispatch mechanics.** Look the command up in the registry. Pre-hello only a
  handler with `available_before_hello` (just `hello`) is accepted; anything
  else ends the handshake. Established, an unknown command or a duplicate
  `hello` is an error reply and the session survives. The Session validates
  nothing command-specific: it calls `handler.execute(ctx, request)`, wraps the
  returned wire result in a `Response(id, result)`, and turns any raise
  (`CommandError`, `protocol.ValidationError`, `GestureError`, or an unexpected
  exception) into an error `Response` — which, pre-hello, also ends the
  handshake (so a version mismatch or bad `hello` params fails cleanly).

### Command handlers — `domain/controllers/commands/`

One `CommandHandler` (`abc.ABC`) per wire command, one file each, mirrored by a
test. `execute(ctx, request)` returns a wire result or raises. Coverage in 7a:

| Handler | Behaviour |
|---|---|
| `ping` | `AckResult`; `resets_inactivity = False`. |
| `echo` | `EchoResult` echoing the payload. (PR 7b.) |
| `pressGesture` | `transcript.gesture(id)` then `ctx.adapter_set.gesture_sender.press(id)`, in order; the first `GestureError` propagates (aborting the rest) and the Session names the id in the error reply. |
| `getSpeech` / `getLastSpeech` / `getNextSpeechIndex` / `waitForSpeech` / `waitForSpeechToFinish` | Delegate to `ctx.speech_buffer`. |
| `getBraille` | Delegates to `ctx.braille_buffer`. |
| `bye` | `ctx.close(CLIENT_BYE)`, return `AckResult` — the Session writes the ack, then honours teardown at the next loop check. |
| `hello` | The bootstrap command (`available_before_hello`); see below. |
| `getFocusInfo` / `getState` / `getConfig` / `setConfig` | One shared `NotImplementedHandler` raising `CommandError` — ports arrive in session E; on the wire a clean command error, distinct from "unknown". |

Wait-style handlers block the session thread for at most their own timeout —
far below the watchdog windows, and the command already reset inactivity, so a
wait can never trip a deadline.

**`hello` (`commands/hello.py`) — the bootstrap.** Wired with the
`AdapterFactory` and NVDA version. On a `protocolVersion` mismatch it raises
`CommandError` *before* touching anything (the factory is never called). Else:
open the transcript, `factory.build(mode)` and install the `AdapterSet` on the
context immediately (so teardown can restore even if a later step raises), make
the buffers (`exact_finish = mode is SILENT`), wire
`speech.set_observer(transcript.speech)`, start both sources, read
`current_synth()`, log `session_opened`, and — silent only — `swap_to_spy()` +
`synth_swapped` (so the transcript reads `SESSION OPEN` before `SYNTH SWAP`).
Returns `HelloResult(protocolVersion, nvda_version, mode, synth, path)`; a
successful pre-hello dispatch transitions the Session to `ESTABLISHED`.

### The registry — `commands/registry.py`

`build_command_registry(factory, nvda_version) -> dict[str, CommandHandler]`:
an explicit hand-written map, read top to bottom (no container, no
auto-registration — Decided). Handlers are stateless singletons; per-session
state is the `SessionContext`. `hello` is constructed with the factory and
version; everything else is dependency-free.

### Teardown — the invariant

One `_teardown` runs exactly once, in `finally`, on every exit path, reading the
context. Each step is individually guarded: if `ctx.adapters` exists, stop
speech source, stop braille source, `restore()` (unconditional, idempotent);
then, if `ctx.swapped_real` is set, `transcript.synth_restored`; then
`transcript.session_closed(reason)`; then close the channel. A raising
transcript or a failing restore can never stop the channel close, and teardown
before `hello` built anything naturally skips the missing pieces.

## Wiring

`wiring.py` at the package root — the composition root, pure and
strict-checked. `build_session(transport, factory, logs_dir, nvda_version,
*, heartbeat_timeout=..., inactivity_timeout=...) -> Session` stacks the
existing pure adapters: `create_session_log(logs_dir)` → `FileTranscript`,
`JsonLinesChannel(transport)`, `RealClock`, `SessionConfig`. The `Transport`
leaf and the real `AdapterFactory` are parameters precisely because they are
session C's NVDA-facing pieces; `wiring.py` read top-to-bottom remains the
answer to "who connects what".

## Tests

### Fakes — deliberately scriptable, built for reuse

New fakes, one per port, mirroring port files, subclassing the ABCs. Agreed
in conversation: these are not minimal stubs — together they form the
**fake NVDA** that both this entry's unit tests and lane 2's integration
tests drive, so their behaviour is part of this contract:

- `tests/fakes/message_channel.py` — **FakeChannel**: scripts whole messages
  via the existing `ScriptedQueue`/`TIMEOUT_EVENT`/`CLOSED_EVENT`, records
  every response written.
- `tests/fakes/gesture_sender.py` — **FakeGestureSender**: records presses;
  can be scripted to reject ids (`GestureError`), and — the fake-NVDA core —
  accepts a **gesture → speech script** (`{"NVDA+f7": ["Elements list
  dialog"]}`): pressing a scripted gesture feeds those lines into the
  session's speech buffer and fires `notify_finished`, so the causal chain
  "press, then read what it said" runs through the wire like real NVDA.
- `tests/fakes/speech_source.py` — **FakeSpeechSource**: keeps the buffer it
  was started with and exposes `emit(text)` for speech *not* caused by a
  gesture (background chatter, focus noise); records start/stop.
- `tests/fakes/braille_source.py` — **FakeBrailleSource**: same shape,
  `emit(cells)` into the braille buffer.
- `tests/fakes/synth_swapper.py` — **FakeSynthSwapper**: records the
  swap/restore call order; a variant raises on restore.
- `tests/fakes/transcript.py` — **FakeTranscript**: records events in memory.
- `tests/fakes/adapter_factory.py` — **FakeAdapterFactory**: records the mode
  it was asked for; returns a prepared `AdapterSet` and hands the test the
  same fake instances it gave the Session.

Because the gesture→speech script needs the session's buffer, the
`FakeAdapterFactory` wires its `FakeGestureSender` to the buffer at `build`
time (the same moment the real factory binds the spy synth to it).

### Unit tests (PR 7a)

The dispatch split makes the tests split the same way — this is the payoff, not
an afterthought:

- `tests/unit/domain/controllers/test_session.py` tests only the **lifecycle +
  dispatch mechanics**: handshake, the two watchdogs, teardown's
  restore-on-every-path, and the mechanics (unknown command, a handler raising,
  the pre-hello gate, `resets_inactivity`). Mechanics use `FakeCommandHandler`
  registries so they don't depend on any real command; lifecycle uses the real
  registry with a `FakeAdapterFactory`. It uses a **builder helper**
  (`run_session(...)`), not fixtures, per AGENTS.md.
- `tests/unit/domain/controllers/commands/test_*.py` — one per handler, each
  exercising `execute` against a **hand-built `SessionContext` with no Session
  and no run loop**. `tests/support/context.py` (a new non-fake support
  package) builds the context from fakes; `test_registry.py` guards that every
  wire command is registered and that the two policy flags sit on exactly the
  right handlers; `test_session_context.py` covers `close` delegation and the
  assert-before-install accessors.
- New fake: `tests/fakes/command_handler.py` — **FakeCommandHandler** (canned
  result or raise; settable policy flags), the double the dispatcher mechanics
  test against.

### Wire-level scenario (PR 7b) — the headless bridge recipe

- `tests/fakes/loopback_transport.py` — **LoopbackTransport**: a connected
  pair of `Transport` implementations over thread-safe queues (each end's
  writes become the other end's reads, with poll timeouts). The last piece
  needed to run the real channel stack without a socket.
- `tests/unit/test_wiring.py` — `build_session` composes the stack from pure
  parts (fake transport + fake factory; a written message appears JSON-framed
  on the transport; the transcript lands under `tmp_path`).
- `tests/integration/test_wire_session_roundtrip.py` — **the recipe, named as
  a use case**: `build_session` over one end of a `LoopbackTransport` with a
  `FakeAdapterFactory`, `run()` on a thread; the test plays the agent on the
  other end, speaking raw protocol bytes. It proves, through real framing and
  real dispatch: `hello` handshake; `echo` round-tripping awkward payloads
  (unicode, nesting, numbers) byte-exactly; scripted gesture → speech →
  `waitForSpeechToFinish` → `getSpeech`; `bye` → teardown with the fakes'
  synth restored. This module is the template lane 2's server↔bridge
  integration tests build on. (AGENTS.md currently describes
  `tests/integration/` as live-NVDA only; PR 7b amends that sentence — the
  directory is for use-case-named end-to-end scenarios, and headless ones run
  in CI while live-NVDA ones remain milestone 6.)

## Delivery plan — two PRs

Sized honestly, one PR would carry five ports, the controller, seven fakes,
the wire change, wiring, the loopback transport, and three test modules —
roughly a thousand-plus lines and two distinct review concerns. That breaks
the house rule (short PRs: one component + its ports + tests), so the entry
ships as two, sequential in lane 1:

- **PR 7a — the component**: the five ports, `Session` + `SessionConfig` +
  `TeardownReason`, the **command-handler layer** (the `CommandHandler` ABC,
  `SessionContext`, the registry, and one handler per v1 command), the fakes,
  and the split test suite (dispatcher/lifecycle + per-handler). One reviewable
  question: *is the controller correct against its ports?* (The command-handler
  decomposition was adopted mid-7a; see "The Session controller and its command
  handlers".)
- **PR 7b — the surface**: `echo` (shared protocol + tests + sync),
  `wiring.py`, `LoopbackTransport`, `test_wiring.py`, the wire-level scenario
  module, and the AGENTS.md integration-tests wording amendment. One
  reviewable question: *does the assembled stack behave over the wire?*

7a merges before 7b starts (strict order within a lane). ROADMAP entry 7 is
split into 7a/7b matching this plan; each PR flips its own sub-entry, and 7b
closes session B.

## Acceptance criteria

Automated, on the `bridge` CI job (plus `shared`, `server`, and the checklist
gate staying green).

PR 7a:

1. Valid silent `hello` → factory built with `SILENT` (only after `hello`),
   synth swapped, both sources started with the session's buffers,
   `HelloResult` carries the negotiated mode, real synth name, NVDA version,
   and the transcript's path; transcript records open and swap.
2. Valid live `hello` → `swap_to_spy` never called; `HelloResult.synth` is
   `current_synth()`.
3. Version mismatch → error naming both versions and teardown; the factory is
   never called, so there is no swap to undo and teardown runs cleanly to the
   channel close (the "restore on every path" guarantee is proven separately in
   criterion 10, where a swap did happen).
4. First message not `hello` (or garbage, or nothing before the heartbeat
   window) → `HANDSHAKE_FAILED` teardown.
5. Speech commands answer from the buffer with correct wire shapes and index
   bookkeeping; `getBraille` likewise; a scripted gesture's speech is
   readable afterwards (the fake-NVDA chain, at the port level).
6. `pressGesture` presses in order and logs each gesture; a `GestureError`
   mid-list aborts the remainder, the error response names the id, and the
   session survives.
7. `ping` resets heartbeat but not inactivity: pings alone for the inactivity
   window → `INACTIVITY_TIMEOUT` teardown; total silence for the heartbeat
   window → `HEARTBEAT_TIMEOUT` teardown (both proven with the fake clock, no
   real waiting).
8. `bye` → ack written, then `CLIENT_BYE` teardown; `ChannelClosed` →
   `CHANNEL_CLOSED` teardown.
9. Mid-session garbage gets an error response (or a transcript note when no
   id is recoverable) and the session continues; a handler exception becomes
   an error response and the session continues; duplicate `hello` and unknown
   commands error without killing the session.
10. **Restore runs on every teardown path** — including when the transcript
    or a source's `stop` raises, and restore's own failure never prevents the
    channel from closing. The fake swapper proves call order and idempotent
    double-restore.
11. `request_teardown` from another thread ends the loop with its reason.
12. Each command handler is exercised through `execute` against a hand-built
    `SessionContext` with **no Session and no run loop**; the registry test
    proves every wire command is registered and that `available_before_hello` /
    `resets_inactivity` sit on exactly `hello` / `ping`.
13. pyright strict is clean; nothing imports NVDA; nothing joins the ignore
    list.

PR 7b:

13. `echo` round-trips `EchoParams`/`EchoResult` in the shared tests, and
    byte-exactly through the wire scenario for unicode / nested / numeric
    payloads.
14. `build_session` composes the production stack from pure parts.
15. The wire scenario passes end to end on a thread: handshake, echo,
    scripted gesture → speech read-back, `bye` with restore — through real
    framing, no socket, no NVDA.
16. Criterion 13 (pyright strict, no NVDA imports) holds for the new files too.

No manual NVDA checklist: neither PR has an NVDA-facing surface. (Session C's
checklist will exercise this controller against live NVDA.)
